Gavigo API

Gavigo API is a Node.js / Express backend that groups several product APIs under one server. All JSON endpoints return a consistent envelope; protected routes require a JWT in the Authorization header.

Response format

Successful responses use HTTP 2xx with:

{
  "success": true,
  "data": { ... }
}

Errors use HTTP 4xx / 5xx with:

{
  "success": false,
  "error": {
    "code": "...",
    "message": "...",
    "status": 400
  }
}

Authentication & headers

Authorization
Bearer <access_token> — required on routes marked auth.
Accept-Language / ?lang=
Disney API routes accept en-US or pt-BR (default en-US). Marked lang.
Content-Type
application/json for POST/PUT bodies unless uploading files.
Role-restricted routes require admin or moderator in addition to a valid token.

Auth /auth

User registration, login, token refresh, and session management.

Method Path Access Description
POST /auth/signup Public Register — request body.
POST /auth/login Public Login — request body.
POST /auth/refreshToken Public Refresh token — request body.
POST /auth/logout auth Logout — request body.
GET /auth/me auth Current authenticated user.

POST request bodies

POST /auth/signup

Registers a new user. Returns accessToken and refreshToken on success.

FieldTypeRequiredDescription
email string Yes Valid email address. Must be unique; duplicate emails return 409.
password string Yes Minimum 6 characters. Stored hashed server-side.
name string No Display name shown on the user profile.
phone string No Contact phone number.
nickname string No Public handle or username. 
{
  "email": "user@example.com",
  "password": "secret123",
  "name": "Jane Doe",
  "nickname": "jane"
}

POST /auth/login

Authenticates an existing user. Returns a new accessToken and refreshToken.

FieldTypeRequiredDescription
email string Yes Registered email address.
password string Yes Account password (minimum 6 characters). 
{
  "email": "user@example.com",
  "password": "secret123"
}

POST /auth/refreshToken

Issues a new short-lived accessToken using a valid refresh token.

FieldTypeRequiredDescription
token string Yes Refresh token received from /auth/signup or /auth/login. Expires after 7 days. 
{ "token": "<refresh_token>" }

POST /auth/logout

Revokes the refresh token server-side. Requires auth header plus the body below.

FieldTypeRequiredDescription
token string Yes Refresh token to invalidate. 
{ "token": "<refresh_token>" }

Users API /users-api

Administrative user management. All routes require auth and admin | moderator.

Method Path Description
GET /users-api/ List users (supports query options).
GET /users-api/findById/:id Get user by ID.
GET /users-api/findByEmail Find user by email query param.
PUT /users-api/update/:id Update user.
DELETE /users-api/delete/:id Delete user.

Disney API (Disneydle) /disney-api

Disney movie catalog and daily guessing game. Public read routes use lang; writes require auth + admin | moderator.

Method Path Access Description
GET /disney-api/ lang List movies.
GET /disney-api/findById/:id lang Movie by ID.
GET /disney-api/autocomplete?text= lang Autocomplete search.
GET /disney-api/movieOfDay?date= lang Movie of the day for a date.
GET /disney-api/timeRemaining lang Time until next daily reset.
POST /disney-api/guess lang Submit a guess — request body.
POST /disney-api/create auth + role Create movie — request body.
PUT /disney-api/update/:id auth + role Update movie.
DELETE /disney-api/delete/:id auth + role Delete movie.
POST /disney-api/uploadImage/:id auth + role Upload poster — multipart body.

POST request bodies

POST /disney-api/guess

Compares your guess against today’s movie (or the movie for the requested date). Send lang via header or query. Returns per-field hints (correct, partial, wrong, etc.).

FieldTypeRequiredDescription
id string Yes MongoDB _id of the movie the player is guessing (from autocomplete or catalog). 
{ "id": "507f1f77bcf86cd799439011" }

POST /disney-api/create

Creates a new Disneydle movie entry. Requires auth and admin | moderator.

FieldTypeRequiredDescription
date string Yes Release year or date label used in guess comparisons (e.g. "1994").
decade string Yes Decade bucket shown in hints (e.g. "1990s").
animation_style string Yes Animation technique (e.g. "CGI", "Traditional").
box_office string Yes Box-office value as a string (e.g. "$1.05B"). Used for greater/less hints.
translations object Yes Localized movie data for the guessing game (see nested fields below).
translations.title string Yes Movie title in the active locale.
translations.cover_image string Yes URL or path to the poster image returned in guess results.
translations.main_characters string[] Yes Names of main characters (partial match supported in guesses).
translations.main_character_types string[] Yes Character archetypes (e.g. princess, animal sidekick).
translations.location string Yes Primary setting or kingdom.
translations.genre string[] Yes Genre tags (e.g. ["Musical", "Adventure"]). 
{
  "date": "1994",
  "decade": "1990s",
  "animation_style": "Traditional",
  "box_office": "$968M",
  "translations": {
    "title": "The Lion King",
    "cover_image": "/img/movies/en-US/lion-king.jpeg",
    "main_characters": ["Simba", "Mufasa", "Scar"],
    "main_character_types": ["Lion", "Hyena"],
    "location": "Pride Lands",
    "genre": ["Musical", "Adventure"]
  }
}

POST /disney-api/uploadImage/:id

Uploads a poster image for an existing movie. Use multipart/form-data, not JSON.

FieldTypeRequiredDescription
file file Yes Image file in the multipart body. Stored in Google Drive; updates the movie’s image fields.

Gemini /gemini

Google Gemini proxy for chat and token counting. Public endpoints.

Method Path Description
POST /gemini/message Chat — request body.
POST /gemini/countTokens Count tokens — request body.

POST request bodies

POST /gemini/message

Proxies a prompt to Google Gemini and returns the model response.

FieldTypeRequiredDescription
contents string Yes Prompt or message text sent to the model.
config object Yes Generation options passed to the Gemini SDK (e.g. temperature, maxOutputTokens, safety settings).
model string No Gemini model ID. Defaults to gemini-2.5-flash when omitted. 
{
  "contents": "Explain JWT in one paragraph.",
  "config": { "temperature": 0.7 },
  "model": "gemini-2.5-flash"
}

POST /gemini/countTokens

Estimates token usage for a given prompt before calling the model.

FieldTypeRequiredDescription
contents string Yes Text to measure (same format as /gemini/message).
model string No Model used for counting. Defaults to gemini-2.5-flash. 
{
  "contents": "Long document text to measure..."
}

Minha Grana API /minha-grana-api

Personal finance: categories, transactions, and billing plans. All routes require auth.

Categories /minha-grana-api/category

Method Path Description
GET /minha-grana-api/category/ List categories.
GET /minha-grana-api/category/autocomplete?text= Autocomplete by name.
GET /minha-grana-api/category/findById/:id Category by ID.
GET /minha-grana-api/category/findByName/:name Category by name.
POST /minha-grana-api/category/create Create category — see request body.
PUT /minha-grana-api/category/update/:id Update { "name"?: "..." }.
DELETE /minha-grana-api/category/delete/:id Delete category.

POST /minha-grana-api/category/create

Creates a spending/income category scoped to the authenticated user.

FieldTypeRequiredDescription
name string Yes Category label (e.g. "Groceries", "Salary"). Used in autocomplete and transaction linking. 
{ "name": "Groceries" }

Transactions /minha-grana-api/transaction

Method Path Description
GET /minha-grana-api/transaction/ List transactions.
GET /minha-grana-api/transaction/findById/:id Transaction by ID.
POST /minha-grana-api/transaction/create Create transaction — see request body.
PUT /minha-grana-api/transaction/update/:id Update transaction.
DELETE /minha-grana-api/transaction/delete/:id Delete transaction.

POST /minha-grana-api/transaction/create

Records an income or expense for the authenticated user. categoryId must reference an existing category owned by the same user.

FieldTypeRequiredDescription
type string Yes Transaction direction: "income" (money in) or "expense" (money out).
name string Yes Short description shown in lists (e.g. "Supermarket").
amount number Yes Monetary value. Use a positive number; sign is implied by type.
date string Yes When the transaction occurred (ISO 8601 date string, e.g. "2025-05-20").
paymentMethod string Yes How it was paid: "credit", "pix", or "cash".
dueDate string Yes Due or settlement date (ISO 8601). For credit expenses, typically the card due date.
categoryId string Yes MongoDB _id of the category from /category/create. 
{
  "type": "expense",
  "name": "Weekly groceries",
  "amount": 245.90,
  "date": "2025-05-20",
  "paymentMethod": "credit",
  "dueDate": "2025-06-10",
  "categoryId": "507f1f77bcf86cd799439011"
}

Billing /minha-grana-api/billing

Method Path Description
GET /minha-grana-api/billing/ List billing records.
GET /minha-grana-api/billing/findById/:id Billing by ID.
POST /minha-grana-api/billing/create Create billing plan — see request body.
PUT /minha-grana-api/billing/update/:id Update billing plan.
DELETE /minha-grana-api/billing/delete/:id Delete billing plan.

POST /minha-grana-api/billing/create

Defines a recurring monthly budget plan with expected amounts per category.

FieldTypeRequiredDescription
name string Yes Plan title (e.g. "May 2025 budget").
bill array Yes Line items mapping categories to expected spending. At least one entry recommended.
bill[].categoryId string Yes Category _id this line belongs to.
bill[].expectedAmount string Yes Planned amount for that category (stored as string, e.g. "500.00").
icon string No Icon name or URL for UI display.
description string No Longer notes about the billing plan. 
{
  "name": "May 2025 budget",
  "icon": "wallet",
  "description": "Monthly household plan",
  "bill": [
    { "categoryId": "507f1f77bcf86cd799439011", "expectedAmount": "800.00" },
    { "categoryId": "507f1f77bcf86cd799439012", "expectedAmount": "350.00" }
  ]
}
